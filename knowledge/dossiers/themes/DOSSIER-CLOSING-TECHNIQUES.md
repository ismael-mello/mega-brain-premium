## Framework Principal: 10 Passos do Fechamento Completo

**Fonte:** EDC-FW-012 ^[FRAMEWORKS.yaml:EDC-FW-012]

```
+------------------------------------------------------------------------------+
| SEQUENCIA DE FECHAMENTO (EDC)                                               |
+------------------------------------------------------------------------------+
| 1.  VALOR DE TABELA       — Apresentar e sustentar. Silencio apos anunciar.|
| 2.  SUSTENTACAO           — Provar valor: cases, ROI, comparacoes.         |
| 3.  PACTO DE ORCAMENTO    — "Se couber no orcamento, fecha hoje?"          |
| 4.  VALOR PROTAGONISTA    — Preco especial + justificativa CAF.            |
| 5.  DUPLA CONFIRMACAO     — Confirmar entendimento antes de cobrar.        |
| 6.  CARTAS NA MANGA       — Quedas, bonus extras, parcelamento diferente.  |
| 7.  TIRAR DA MESA         — Fingir encerrar negociacao para baixar guarda. |
| 8.  DOWNSELL              — Produto menor se cliente verdadeiramente nao   |
|                             pode.                                           |
| 9.  SINAL DE COMPROMISSO  — Ultimo recurso: minimo R$1.000 reembolsavel.  |
| 10. PEGANDO O PAGAMENTO   — Enviar link, aguardar comprovante na call.     |
+------------------------------------------------------------------------------+
```

## Framework: 6 Passos de Contorno de Objecao

**Fonte:** EDC-FW-005 ^[FRAMEWORKS.yaml:EDC-FW-005]

```
+------------------------------------------------------------------------------+
| 6 PASSOS — CONTORNO DE OBJECAO (EDC)                                       |
+------------------------------------------------------------------------------+
| 1. DESPRESSURIZAR   — "Eu entendo perfeitamente." Concordar e elogiar.    |
| 2. TIE-DOWN/AMARRA  — Amarrar a ponto de concordancia anterior.            |
| 3. ISOLAR           — "De 0 a 10, o que te impede?" Objecao real.         |
| 4. CONFIRMACAO DUPLA— Confirmar que e ESSA a unica objecao.               |
| 5. LIDAR            — Mudar angulo, usar metafora, reframe.               |
| 6. PEDIR PELA VENDA — "Entao podemos seguir?"                             |
+------------------------------------------------------------------------------+
| REGRA: Nao interromper. Usar silencio. Falar devagar.                      |
+------------------------------------------------------------------------------+
```

## Framework: CAF (Custos Administrativos e Financeiros)

**Fonte:** EDC-FW-013 ^[FRAMEWORKS.yaml:EDC-FW-013]

```
NARRATIVA CAF — JUSTIFICATIVA DE DESCONTO SEM DESVALORIZAR:
1. Sustentar 100% o valor de tabela
2. Fazer pacto de fechamento em call
3. "Normalmente pessoas demoram 30 dias para decidir"
4. "Esses 30 dias tem custo comercial, admin e financeiro (CAF)"
5. "Para pessoas decididas como voce, pego esse custo e dou como bonus"
6. Revelar valor protagonista

> "Nao e desconto porque sou legal. E economia que eu passo pra voce."
```

## Sistema de Pactos (Microfechamentos)

**Fonte:** EDC-FW-004 ^[FRAMEWORKS.yaml:EDC-FW-004]

```
SEQUENCIA DE PACTOS (do inicio ao fim da venda):
- PACTO DE INICIO      — Pos introducao, antes do questionario
- PACTO SIM OU NAO     — Pos SPIN, compromisso de decidir na call
- PACTOS CREDIBILIDADE — Durante pitch, confirmacoes de entendimento
- PACTO DE ORCAMENTO   — "Se caber, fecha hoje?"
- PACTO FINAL          — Antes do preco, compromisso imediato

"O que e um pacto? E um micro fechamento sem pegar dinheiro."
```

## Ancoragem de Valor (Contraste)

**Fonte:** EDC-FW-006 ^[FRAMEWORKS.yaml:EDC-FW-006]

```
1. ANCORAGEM ALTA    — Quanto custaria fazer sozinho ou por outro caminho
2. VALOR DE TABELA   — Preco "normal" do produto/servico
3. SUSTENTACAO       — Provar que vale: cases, ROI, comparacoes
4. VALOR PROTAGONISTA— Preco especial para decisao em call
5. CAF               — Justificar: economia CAF que repasso pra voce

> Gatilho mental: contraste faz o preco final parecer acessivel.
```

## Tecnicas Avancadas de Fechamento

### BITs (Pre-Pitch)
**Fonte:** EDC-FW-014

```
B - BELIEF    (Crenca):    Cliente acredita que a solucao funciona
I - IDENTITY  (Identidade): Cliente se ve como alguem que merece/precisa
T - TRUST     (Confianca): Cliente confia no closer e na empresa
```

### Por Tras dos Panos
**Fonte:** EDC-FW-015

```
Apos interesse demonstrado:
- Compartilhar tela: CRM, contrato, plataforma
- "Vou te mostrar como funciona por tras dos panos"
- Preencher dados do cliente enquanto conversa
- Naturaliza o processo de compra, reduz ansiedade
```

### Escolha 2 de 3 Bonus
**Fonte:** EDC-FW-016

```
1. Apresentar 3 bonus valiosos
2. "Voce pode escolher 2 dos 3"
3. Cliente instintivamente quer os 3
4. Oferecer os 3 como "condicao especial"
5. Cliente sente que ganhou mais do que esperava
```

```
+------------------------------------------------------------------------------+
| VENDA DE IMPACTO — TIMELINE                                                 |
+------------------------------------------------------------------------------+
| INTRODUCAO     5 min  — Quebra-gelo, FORM, rapport, marketing pessoal      |

| APRESENTACAO   ~20 min — Empresa → Metodo → Produto (EDC-FW-009)           |

| TOTAL:         40-45 min B2C                                               |
+------------------------------------------------------------------------------+
| META: 1 fechamento em cada 3-4 calls (25-40%)                              |
+------------------------------------------------------------------------------+
```

## Caso Especial: 2a Call (Falta Decisor)

**Fonte:** EDC-FW-017

```
REGRA: NUNCA mostrar preco sem todos os decisores.
1. Fazer apresentacao completa normalmente
2. "Para montar a melhor proposta, preciso falar com voce e [decisor]"
3. Agendar segunda call com todos
4. PIX de honra/compromisso (reembolsavel) para garantir retorno
5. Na segunda call: focado em fechamento (apresentacao ja foi feita)
```
